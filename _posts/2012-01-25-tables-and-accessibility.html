WCAG 2.0 says you can use tables still for layout but it is advisable to use CSS for layout.

For data tables a summary tag can be used to provide information on complex tables for screenreaders.

A caption tag is used for the table title.

th and td tags are used for table header and table data entries.

Scope should be added if the table is more complex ie. not just headers in first row or column.

Id and header attributes can be used for still more complex tables.

Tags can be used for grouping rows - not essential. This does enable for scrolling of the table body and when printed the table headers and footer will print over the pages for long tables.

Can also provide css hooks for styling. Note tfoot comes before tbody
and there can be multiple tbodyÂ tags. Colgroup and col can be used to group columns. Again not essential.
