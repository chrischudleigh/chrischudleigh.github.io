Hover intent is a jquery plugin which causes dropdown menus to only activate when the user slows down as if to stop on the menu.

It overrides the hover function.

It also has a timeout function so menus don't fly shut like css menus.

How it overides css menus is the question - ie. this would mean the menu uses jquery and not css if hoverintent is used?

 http://www.thatsquality.com/articles/creating-delayed-drop-down-menus-in-jquery-without-losing-accessibility 

Actually it may be better to do the following:

$('#menu li ul').css({ display: "none", left: "auto"}); $('#menu li').hover(function() {

$(this) .find('ul') .stop(true, true) .slideDown('fast');}, function() { $(this)

.find('ul') .stop(true,true) .fadeOut('fast');});

and replace hover with hoverintent.

Of course display none is better replaced with absolute positioning for accessibility or the new technique which I cannot quite remember.
