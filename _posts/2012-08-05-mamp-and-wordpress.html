The local copy of wordpress stopped working for me. When I got a fibre connection with a new isp, I got a new ip and so wordpress was looking for the old ip address I had.

Fix: Change the mysql database entry in wordpress for the url.

While investigating, I decided it is better to create a virtual host eg. blog.local This is added in two places:

See http://sawmac.com/mamp/virtual/  for details.

If you place the website in your sites directory, wordpress also needs you to give the directory admin access it seems.

To duplicate the remote site:

  • Export and import the content
  • Redo the widgets and the menus.
